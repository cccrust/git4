# git4 - 開發紀錄 v0.2 (規劃中)

## v0.2 規劃目標

在 v0.1 中，我們完成了最初始的底層機制，具備了產生 Blob、Tree 與 Commit 物件的功能並能夠印出 log。為了讓版控系統更加實用，v0.2 我們將重點放在**分支系統 (Branching)** 與 **檔案復原 (Checkout)**。

### 階段一：分支支援 (Branch)
- `git4 branch`: 列出全部現存的分支。
- `git4 branch <name>`: 建立一個新的分支指標，將 `refs/heads/<name>` 指向當前的 `HEAD`。
- 分支指標結構將完全依賴 `refs/heads/` 資料夾，這能幫助開發者多線管理。

### 階段二：切換與復原 (Checkout)
- `git4 checkout <branch/commit>`: 將 HEAD 轉移到對應的目標。
- **檔案還原機制**：當切換 commit 時，需要透過讀取該 commit 的 Tree，把當下的檔案系統寫入、覆蓋回工作目錄，具備真正的「還原檔案」的能力！
- 此機制需要實作 `read-tree` 等內部方法，遞迴解析 Tree 內的 blob 並寫入到真實磁碟上。

### 階段三：增強測試腳本
- 透過 `test.sh` 加入建立分枝與切換 checkout 的實驗流程，確保還原出的檔案內容正確無誤。
